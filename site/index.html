読みが壊れにくい順

LLM に自由に読ませるのではなく、候補ラティスの内側だけで決めます。拡張のローカル変換と同じ考え方です。

  1. 個人辞書を最優先 リクエストの user_dict、拡張のクリック学習が最上位です。
  2. 慣用句は trust 「一日中」「何度も」など、崩れやすい句を trust / 固定読みで守ります。
  3. ラティス＋文脈キュー 形態素と辞書から候補を作り、近くの語句で読みを選びます。候補外は捨てます。
  4. 低確信は辞書へ戻す 確信度が低いときは base 読みに倒し、変な読みを字幕に載せにくくします。
