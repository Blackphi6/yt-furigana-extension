# プライバシーポリシー

最終更新: 2026-07-22  
対象: YT Furigana（Chrome 拡張および関連のローカル／任意サーバー機能）

本ポリシーは、本ソフトウェアの利用に関する個人情報・端末データの取り扱いを説明します。法的助言ではありません。

## 1. 運営

- リポジトリ: https://github.com/Blackphi6/yt-furigana-extension
- 連絡: GitHub Issues / リポジトリに記載の連絡手段

## 2. 収集するデータ（Free / 既定）

**既定（Free）では、開発者のサーバーへ字幕本文や学習データを送信しません。**

| データ | 保存場所 | 目的 |
|--------|----------|------|
| 設定（エンジン種別・URL など） | `chrome.storage.sync`（Google アカウント同期の範囲） | 設定の保持 |
| ライセンスキー・読み API キー | `chrome.storage.local`（端末内。Google 同期しない） | 認証 |
| ユーザーが選んだ読み辞書 | `chrome.storage.local` | ふりがな学習 |
| 学習ログ（字幕断片・ページ URL） | `chrome.storage.local` に**自動蓄積**（上限あり）。ポップアップで**オプトアウト可**。書き出しはユーザー操作時のみ | 辞書改善の素材 |
| 共有読みパック（受信） | 公開読みエンジンから取得し `chrome.storage.local` に保存 | 集計／公開済みの **表層→読み** のみ。字幕本文は含まない。既定オン・ポップアップでオフ可。Premium 同期用 URL には引きずられない |

字幕テキストの解析は、原則としてユーザーのブラウザ内（Kuromoji / Sudachi 等）で行います。

### YouTube / TVer 上での読み取り

- 画面上の字幕 DOM を読み、ルビを重ねます。
- コメント・視聴履歴・説明文などは収集しません。
- 既定の再生パスでは YouTube timedtext / 字幕配信 API を取得しません（DOM のみ）。

## 3. 任意機能で送信されうるデータ

ユーザーが自ら有効化・接続した場合に限ります（共有読みパックの**受信**は §2 の既定オン・オフ可）。

| 機能 | 送信先 | 内容 |
|------|--------|------|
| 読みAPI（BYO） | ユーザーが指定した URL | 字幕テキスト（選択時はトラック全行のこともあり）、個人読み辞書（`user_dict`） |
| Ollama | ユーザーのローカル Ollama | 変換用テキスト |
| Premium 辞書同期 | ユーザーが指定した同期サーバー | 読み辞書、ライセンスキー |
| Premium サーバー共有辞書取得 | 同上 | 認証トークン（手動ボタン） |
| 匿名の読み訂正（オプトイン） | 公開読みエンジン（または指定 URL） | 表層・読み・短い前後文脈のみ（動画 URL なし） |
| デモの読み提案（任意） | 公開読みエンジン | 固定リストの表層→読みのみ。**審査キューに保管**し、即時にはみんな用パックへ載せない。字幕全文・動画 URL は送らない |
| Premium 購入（サイト） | Stripe Checkout および読みエンジン側 | 決済は Stripe。メールは任意入力時のみ |

**第三者の公開 API を指定した場合、その事業者のプライバシーポリシーが適用されます。** 本プロジェクトは当該サービスの管理者ではありません。

公開読みエンジンは不正利用防止のため、接続元 IP 単位のレート制限をかけることがあります（個人の特定目的ではありません）。
匿名の読み訂正は、同一接続からの重複票をまとめるため投票者ID（IPの不可逆ハッシュ）をサーバー内のみで用います。公開パックにはIPを含めません。
デモサイトからの読み提案も同様に投票者IDを付与して審査キューに保管します。ヒューリスティックのあと、設定時は LLM が非同期で振るい分けます。運営が curated に載せるまで Free パックには反映しません。

## 4. 広告・トラッキング

本拡張の既定実装に、広告 SDK や行動トラッキングは含みません。

## 5. 子どもの利用

13 歳未満を対象としたサービスではありません。保護者の監督のもとでご利用ください。

EEA / UK 等での追加権利（アクセス・消去等）については、既定データは端末内のみのため拡張の削除または学習ログ消去で対応できます。サーバー側データをセルフホストしている場合は、当該運用者へご連絡ください。連絡先は GitHub Issues です。

## 6. データの削除

- 拡張の削除、または Chrome のサイトデータ／拡張データ削除により、端末内の学習・設定・キーは消去できます。
- ポップアップの「学習ログを消去」、および「曖昧語サンプルを自動蓄積」のオフで、蓄積の停止と既存ログの削除ができます。
- 共有読みパックの受信はポップアップでオフにできます。
- Premium 同期サーバー上のデータは、当該サーバーの運用者（セルフホスト時は利用者自身）の手順に従ってください。
- Stripe 上の決済記録は Stripe およびカード会社のポリシーに従います。

## 7. 変更

本ポリシーは予告なく更新されることがあります。重要な変更はリポジトリの更新日で示します。
