【短い説明】（132文字以内・そのまま貼る）
--------------------------------------------------------------------------------
YouTube / TVer の日本語字幕にひらがなルビ（非公式）。端末内で処理。クリックで読みを覚えます。有料は読み辞書の移動などが任意。

--------------------------------------------------------------------------------
【詳細説明】（そのまま貼る）
--------------------------------------------------------------------------------
YT Furigana は、YouTube と TVer の日本語字幕にひらがなのふりがな（ruby）を重ねて表示する Chrome 拡張です。

【無料】
・この端末だけで動く読み付け（設定不要）
・字幕の漢字をクリックして読みを覚え直す
・迷った言葉のメモも端末内のみ（オフにできます）
・既定ではクラウドへ字幕本文を送りません
・みんなの読みパック（表層→読みのみ）を受け取れる（オフ可）

【有料オプション（任意）】
・直した読み辞書を別のパソコンへ移す
・指定サーバー上の共有辞書を手動で取り込む

ソース: https://github.com/Blackphi6/yt-furigana-extension
サイト: https://blackphi6.github.io/yt-furigana-extension/

字幕・映像の著作権は各権利者に帰属します。本拡張は画面上の表示加工であり、コンテンツの再配布を目的としません。

【単一用途の説明】（Dashboard「単一用途の説明」にそのまま貼る）
--------------------------------------------------------------------------------
YouTube と TVer のページ上に表示されている日本語字幕（画面上の字幕 DOM）へ、ひらがなのふりがな（ルビ）を重ねて表示することだけを目的とします。字幕の漢字をクリックして読みを覚え直す学習機能、および任意の読み辞書まわり（端末内保存・オフ可能な共有読みパック受信・ユーザーが自ら有効化した同期など）は、いずれもこのふりがな表示を正確・便利にするための付随機能です。字幕・映像の再配布や、単一目的と無関係なデータ収集は行いません。本拡張は Google / YouTube / TVer の公式機能ではありません。

--------------------------------------------------------------------------------
【storage が必要な理由】（そのまま貼る）
--------------------------------------------------------------------------------
設定（ふりがなのオンオフ、エンジン種別、共有読みパック受信の可否など）と、ユーザーが字幕上で選んだ読み辞書・学習メモを端末内に保存するために storage を使います。chrome.storage.sync / local に保持し、ふりがな表示の継続と読みの覚え直しにのみ使用します。広告やトラッキング目的では使いません。

--------------------------------------------------------------------------------
【declarativeNetRequest が必要な理由】（そのまま貼る）
--------------------------------------------------------------------------------
ユーザーが任意でローカルの Ollama（localhost:11434）を読み推定に使う場合に限り、拡張からローカル API へリクエストできるようにするためです。ブラウザが付与する Origin ヘッダーがローカルサーバー側で拒否されることがあるため、該当ホスト向けに Origin を調整する declarativeNetRequest ルールを使います。YouTube / TVer の字幕取得や一般サイトの通信改変には使いません。通常のふりがな表示（端末内エンジン）ではこの経路は不要です。

--------------------------------------------------------------------------------
【declarativeNetRequestWithHostAccess が必要な理由】（そのまま貼る）
--------------------------------------------------------------------------------
上記の Ollama 向け declarativeNetRequest ルールは、対象ホストへのアクセス許可があるときにのみ適用される MV3 の仕組みです。ユーザーがローカル Ollama 接続を使うときにホスト許可と合わせて動作させるために declarativeNetRequestWithHostAccess を要求しています。任意機能であり、未使用時に第三者サイトの通信を改変する目的ではありません。

--------------------------------------------------------------------------------
【ホスト権限 が必要な理由】（そのまま貼る）
--------------------------------------------------------------------------------
必須ホスト:
・www.youtube.com / tver.jp … 画面上の日本語字幕 DOM にルビを差し込むため（通常再生では YouTube timedtext API は使いません）。
・yt-furigana-readings.onrender.com … Free の共有読みパック（表層→読みのみ）を受信するため。オフにできます。ユーザーがオプトインした匿名の読み訂正送信先にもなります。字幕本文の収集・再配布はしません。

任意ホスト（ユーザー操作時のみ）:
・localhost / 127.0.0.1 の Ollama・読みエンジン … ローカル接続を選んだときだけ許可ダイアログ。
・http(s)://*/* … Premium の辞書同期など、ユーザーが指定したサーバー URL に接続するときだけ。

ホスト権限は単一目的（字幕へのふりがな表示とそのための読みデータ）以外には使いません。

Unofficial accessibility tool for Japanese captions on YouTube and TVer.
"YT" in the product name is not a YouTube trademark.
No affiliation with Google, YouTube, or TVer.

☐ ユーザーのアクティビティ … クリック監視・キーロガー等はしない
