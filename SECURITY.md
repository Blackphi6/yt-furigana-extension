# Security Policy

## Supported versions

Please use the latest commit on the default branch when reporting issues.

## Reporting a vulnerability

- Prefer **GitHub Security Advisories** (private) on this repository when available.
- Otherwise open a GitHub Issue **without** including secrets, private keys, or personal data dumps.
- Do not file public issues that contain user subtitle corpora or API keys.

## Scope notes

- Free mode keeps learning data on-device by default.
- BYO reading API / Premium sync send data only to endpoints the user configures.
- Dictionary binaries (Kuromoji / Sudachi) are third-party; report upstream issues to those projects when appropriate.
